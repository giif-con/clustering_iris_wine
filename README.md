# Clustering: учебные эксперименты

Учебный проект по кластеризации на классических датасетах из `sklearn`.
Цель — познакомиться с методами без учителя, научиться подбирать гиперпараметры и оценивать качество кластеризации.

## Стек

- Python 3
- NumPy, Matplotlib
- scikit-learn (`KMeans`, `DBSCAN`, `StandardScaler`, `silhouette_score`)
- Jupyter Notebook

## Что сделано

### Эксперимент 1. Iris (150 объектов, 4 признака)

- **KMeans**: перебор числа кластеров `n_clusters` от 2 до 5, оценка по silhouette.
- **DBSCAN**: сеточный перебор параметров `eps` и `min_samples`, учёт доли шумовых точек.
- Визуализация кластеров в координатах признаков.

### Эксперимент 2. Wine (178 объектов, 13 признаков)

- Предобработка: стандартизация признаков через `StandardScaler` (нужна из-за разных масштабов).
- **KMeans**: перебор `n_clusters` от 2 до 9 на стандартизованных данных.
- **DBSCAN**: перебор `eps` из `{3.0, 2.5, 2.0, 1.5}` и `min_samples` из `{2, 4, 5, 8}`.

## Результаты

**Iris:**
- KMeans: лучший silhouette — **0.68** при `n_clusters = 2`, при увеличении числа кластеров метрика падает.
- DBSCAN: лучший silhouette — **0.74** (`eps ≈ 0.51`, `min_samples = 8`), но доля шума ≈ 15%.
  При `eps = 1`, `min_samples = 2` — silhouette **0.69** без шума.
- Оба метода сошлись на 2 кластерах, хотя в реальной разметке Iris 3 класса.
  Причина — классы *versicolor* и *virginica* сильно пересекаются по признакам и воспринимаются как один кластер.

**Wine:**
- KMeans на стандартизованных данных даёт разумное разбиение на 2–3 кластера.
- DBSCAN чувствителен к подбору параметров; при слишком маленьком `eps` растёт доля шума.

## Что можно улучшить

- Понизить размерность (PCA / t-SNE) перед визуализацией, чтобы картинки отражали кластеры в полном пространстве признаков, а не в проекции на 2 оси.
- Сравнить кластеризацию с истинной разметкой через ARI / NMI, а не только silhouette.
- Добавить агломеративную (иерархическую) кластеризацию как третий метод.
